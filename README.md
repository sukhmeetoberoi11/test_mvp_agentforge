# SaaS Sales Meeting Scheduler

> Generated by [AgentForge](https://agentforge.io)

## Overview

The SaaS Sales Meeting Scheduler automates meeting scheduling by parsing outbound emails, aligning with calendars, and scheduling meetings through webhooks. The system's core loop involves receiving an email, extracting relevant information, checking calendar availability, and scheduling the meeting. Key components include an email parser, LLM for context understanding, calendar API for availability checks, webhook scheduler for meeting setup, and Redis for session state management. The architecture is designed to handle medium load efficiently, with scalability options such as horizontal scaling of Redis and distributed LLM processing. Security measures include secure authentication, data isolation, and guardrails to ensure appropriate responses.

## Quick start

## Deployment

This repository includes two GitHub Actions workflows — **Azure Container Apps** (default)
and GCP Cloud Run (available but inactive).

### One-time Azure setup (automated on connect)

When you call `POST /github/connect` with AgentForge's Azure admin credentials configured,
AgentForge automatically configures OIDC and writes AZURE_* secrets to the GitHub
**Environment** named in `AZURE_GITHUB_ENVIRONMENT` (default: `production`).

The workflow (`.github/workflows/deploy-azure.yml`) uses `environment: production` and
is triggered only by `POST /deployments/{agent_id}` (or manual **Run workflow** in GitHub).

### GCP Cloud Run (manual only)

A `.github/workflows/deploy-gcp.yml` is present for future use. It does **not** run on
push — only via **Actions → Deploy to Cloud Run → Run workflow**. To use it, add three
GitHub Secrets and set `DEFAULT_DEPLOYMENT_PROVIDER=gcp` in AgentForge:
- `GCP_WORKLOAD_IDENTITY_PROVIDER`, `GCP_SERVICE_ACCOUNT`, `GCP_REGION`
